谈一谈我印象中的互联网团队

Mar 29 2017

  2. 开发阶段
  3. 测试阶段
    1. 基本原则
    2. 如何提测

  • 相关建议

    • ##办公环境
    气氛可以轻松活泼，但是开发任务起来后需要相对安静的环境，要给开发人员良好的思考空间，敲代码的时候有其他同事大声喧哗很影响思维，各个部门（模块）格局分明，不要混搭，产品模块，市场模块，后台模块，前端模块，移动模块，其他模块

    ##开发平台准备

    项目平台 小团队 大团队
    管理平台 tower，coding 自建，readmine，禅道，均有开源
    接口管理平台 在线rap 自建rap
    代码管理平台 coding 自建gitlab
    持续集成 自建jenkins
    成员交流 qq等 内部
    考勤 dingding 内部OA

    大力推荐：
    rap：mock数据功能加快前端开发（定义好接口，即可访问到生成数据），占用内存也不大，阿里云上2G内存机器即可，代价小，效率提升在以上工具中最为明显

    ##项目开发流程

    需求整理

    这个阶段主要是产品经理的工作，或者小团队的集思广益，这个阶段最为关键，公司发展到一定阶段，开发团队是没有瓶颈的，但是一个好的点子／产品经理却很难遇到，不要评估出伪需求，也不要一开始就大而全，需求应该分期规划，先做核心功能，验证模式，这个阶段一般开发leader也会参与

    这个阶段产品要有清晰的思维产物，以及两个书面产物，一个是需求文档，第二个是产品原型

    产品原型又有ps的初步原型，或者专业原型工具的高保真原型（如Axure）下图就是一个Axure页面，有动态交互细节，和像素参考，进阶设计师必备技能

    开发阶段

    1.选型

    语言（默认java）node？php？
    数据库（默认mybatis加上redis）hadoop? mongodb?
    服务器（默认云：传统？docker？）
    开发框架（ssm：传统集成？spring boot？）个人偏向boot+spring cloud
    前后端分离（默认）
    客户端授权验证（没有？）

    2.模块划分
    如用户信息模块，企业信息模块，订单模块，风控模块

    3.数据库建库
    确定字段，这是在2的基础上 数据库工具推荐DataGrip

    4.任务指派
    一般由开发负责人指派，要注意工作的进度，可靠性，也要注意带动新人的发展，配合项目管理工具（个人用过禅道（公司初期），readmine（公司重构后），coding（自己玩），tower（大三团队期间）），一方面是管理层追踪，一方面是员工之间的透明

    任务状态有
    创建 指派给 进行中 已解决 关闭 反馈
    任务优先级
    从高到低
    每个人能看到自己的工作量

    5.代码阶段
    就拿微服务来说，所有微服务应该风格相同，基础配置fork自一份原始代码，不过对小团队不作要求

    6.是否支持上一版系统

    测试阶段

    基本原则

    不同环境之间需要隔离

    如何提测

    1.代码层面
    代码有切换运行环境的“总开关”
    以springboot为例子
    application.yml文件中配置不同的profiles

    以android为例
    build.gradle文件中配置不同的buldeType

    2.发布方式
    人工发布费时费力，还有出错率
    建议采用CI工具，如jenkins绑定git仓库，设置触发器发布／手动登录jenkis构建

    测试反馈

    1.先测试环境测试

    2.后模拟线上环境测试

    3.上线观察期

    4.稳定
    测试出bug后 把问题经项目管理工具指派给开发者

    问题分类有
    缺陷，新增需求，优化

    问题的状态和任务类似
    创建 指派给 进行中 已解决 关闭 反馈

    问题优先级
    从高到低

    发布

    相关建议

    结合实际，以下可以推行

    • 后端开发全面转型IDEA
    • 新项目转向SpringBoot
    • 使用一个项目管理工具
    • 采用git仓库，自建／托管 （4G阿里云／coding）
    • 使用接口管理工具 rap／自建rap （在线使用／2G阿里云）
    • 接口定义统一restfu风格
    • 代码配置区分profiles
    • 代码架构由负责人负责，提供技术支持和所有架构上的代码
    • codereview
    • 负责人本地jenkins发版
